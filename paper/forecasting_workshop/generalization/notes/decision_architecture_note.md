# Decision Architecture Note

This note summarizes the support-only decision-architecture comparison under the frozen-source rules documented for the generalization package. The comparison is narrow: it asks whether the target-versus-executed discrepancy recurs outside the exact current RL interface, while keeping the paper's main wording centered on executed-path evaluation under constrained execution. It does not authorize any claim of universal architectural disagreement.

The architecture-level aggregation uses the same conservative disagreement audit that the master package uses. A row is labeled `ranking_mismatch` when executed-path and target-based evaluation imply different arm ordering once the audit's tie tolerance is applied, `sign_flip` only when the two views point in opposite non-tied directions, and `magnitude_only` only when the ranking stays the same but the target view materially damps the executed-path gap.

The RL-source arm remains the main in-family reference. In `arch_rl_selected`, the zero-cost row stays near-flat at `delta_sharpe_exec=0.0029`, while the positive-cost rows are `+0.0126` at `kappa=5e-4` and `+0.0223` at `kappa=1e-3` on the executed path. Under the conservative audit these positive-cost rows are still classified as `ranking_mismatch`, because the target-based deltas remain too small to preserve the executed-path ordering.

The independent non-RL deadband pair is now one of the central support additions in this package. The selected champion `arch_deadband_partial_champion` stays within the validation-aligned zero-cost qualification screen at `delta_sharpe_exec=0.0078` and then remains positive at `+0.0080` for `kappa=5e-4` and `+0.0082` for `kappa=1e-3`, with `ranking_mismatch` on both positive-cost rows. The paired runner-up `arch_deadband_partial_runnerup` is smaller but qualitatively similar: its zero-cost row is `delta_sharpe_exec=0.0055`, and its highest positive-cost row reaches `+0.0058` while preserving the same disagreement label. These two rows should be read as independent non-RL support because they do not use RL target replay and they keep the target mapping fixed while changing only the execution rule.

The redesigned independent non-RL volatility-scaled pair now adds a second family-level support block. The champion `arch_vol_spike_eta_champion` remains near-flat at zero cost with `delta_sharpe_exec=0.0046`, then stays positive at `+0.0042` for `kappa=5e-4` and `+0.0039` for `kappa=1e-3`, again with `ranking_mismatch` on the positive-cost rows. The paired runner-up `arch_vol_spike_eta_runnerup` is slightly smaller but keeps the same qualitative pattern: its zero-cost row is `delta_sharpe_exec=0.0035`, and its highest positive-cost row reaches `+0.0030` while still classifying as `ranking_mismatch`. This second non-RL family matters because it is independent from both RL replay and deadband gating while still starting from the same shared deterministic target mapping.

`arch_rule_eta_fixed` still reproduces the RL-selected numbers under the current spec, which is expected because it replays the same frozen RL target stream with the same fixed-eta map. It should therefore be kept in the package as an implementation-consistency replay rather than promoted as the main independent support result.

The non-RL linear or proximal support arm is mixed in a different way. In `arch_linear_prox`, the selected positive-cost tau improves the executed-path Sharpe relative to `tau=0` by `+0.7987` at `kappa=5e-4` and `+1.8005` at `kappa=1e-3`, but target-based and executed-based evaluation are numerically aligned there, so the disagreement label is `none`. Its zero-cost row is also not near-flat: `delta_sharpe_exec=-0.2055`. This means the linear/prox comparator is useful as a cost-sensitive execution support arm, but it does not reproduce the target-versus-executed discrepancy.

The safe architecture verdict is therefore stronger than before but still narrow. The RL-source row remains supportive, the independent non-RL deadband champion and runner-up now add two Green support rows, the independent non-RL vol-spike champion and runner-up add another two Green support rows, and the linear/prox arm still marks the limit case where disagreement disappears. The architecture-level verdicts are `Green` for `arch_rl_selected`, `Green` for `arch_deadband_partial_champion`, `Green` for `arch_deadband_partial_runnerup`, `Green` for `arch_vol_spike_eta_champion`, `Green` for `arch_vol_spike_eta_runnerup`, `Green` for `arch_rule_eta_fixed`, and `Yellow` for `arch_linear_prox`. That updated package reduces the specific `RL-only artifact` concern more directly than the earlier architecture draft, because the same qualitative pattern now survives in two independent non-RL execution families as well. It still does not justify claiming universal disagreement across all execution-layer families.
