Many forecasting-driven decision systems stop at proposed targets. In cost-sensitive portfolio decisions, however, realized positions need not equal proposed targets, because trading frictions intervene between what is suggested and what is actually carried into the next return interval. This distinction changes realized turnover, realized costs, and the path that should be evaluated.

We therefore study a forecast-to-execution interface in which target and executed portfolios are separated and primary evaluation is attached to realized executed paths rather than counterfactual target paths. The paper remains deliberately narrow. It does not propose a new RL algorithm, it does not claim a stronger predictive signal, and it does not generalize beyond a portfolio decision case study. The question is simply whether, holding the predictive signal fixed, the forecast-to-execution interface materially changes realized decision quality under costs.

Our central evidence comes from a frozen-policy RL protocol on the canonical U27 split. Validation selects an interior operating point `eta=0.5`, and held-out evaluation compares that selected point against the immediate-execution baseline `eta=1.0` using executed-path net Sharpe as the primary metric. We then use target-versus-executed accounting diagnostics to explain why executed-path evaluation is primary, and we use a regenerated canonical dense-friction diagnostic bundle to show that the implementation effect becomes more consequential as frictions rise. Finally, we add a linear forecast-plus-convex-optimization comparator as auxiliary supporting evidence, including a narrow robustness check around the validation-selected `c=3000` operating point and a conservative same-forecast / different-decision-quality table that stays at the `similar forecasting information` level.
