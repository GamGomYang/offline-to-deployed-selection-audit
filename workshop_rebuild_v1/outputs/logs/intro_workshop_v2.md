Many forecasting-driven systems are assessed at the level of proposed outputs, as if those outputs were directly realized by downstream decision layers. Under frictions or implementation constraints, however, proposed targets need not coincide with realized decisions. This creates an evaluation problem: forecast quality and realized decision quality are not the same object once a forecast-to-decision interface intervenes. The right question is therefore not only whether forecasts look good before implementation, but also whether the system is evaluated on the realized path it actually induces.

We study this question in cost-sensitive portfolio decisions as a narrow case study in which the predictive signal is held fixed and only the forecast-to-execution interface is varied. Primary evaluation is attached to executed-path outcomes rather than target-path quantities, which remain diagnostic only. In the central frozen-policy comparison, the validation-selected interior operating point `eta=0.5` is evaluated against the immediate-execution baseline `eta=1.0`; the positive-cost rows improve, while the zero-cost row remains near-flat. The contribution is therefore an implementation-side evaluation argument, not a stronger-prediction claim and not a new RL-method claim.

Our central evidence comes from a frozen-policy RL protocol on the canonical U27 split. Validation selects an interior operating point `eta=0.5`, and held-out evaluation compares that selected point against the immediate-execution baseline `eta=1.0` using executed-path net Sharpe as the primary metric. We then use target-versus-executed accounting diagnostics to explain why executed-path evaluation is primary, and we use a regenerated canonical dense-friction diagnostic bundle to show that the implementation effect becomes more consequential as frictions rise. Finally, we add a linear forecast-plus-convex-optimization comparator as auxiliary supporting evidence, including a narrow robustness check around the validation-selected `c=3000` operating point and a conservative same-forecast / different-decision-quality table that stays at the `similar forecasting information` level.
